              Schema 标记使用说明（AI 搜索优化）
═══════════════════════════════════════════════════════════════

📌 什么是 Schema？
   Schema 是一种标准化的数据格式，让 AI（ChatGPT、Perplexity、Claude）
   能够准确理解您的内容，并在搜索结果中显示结构化信息。

   就像给内容贴标签，告诉 AI："这是FAQ"、"这是产品参数"、"这是图片"

═══════════════════════════════════════════════════════════════
                    一、现有的 Schema 类型
═══════════════════════════════════════════════════════════════

系统已经支持以下 Schema 类型（自动生成）：

✅ 1. 图片 Schema (ImageObject)
✅ 2. 常见问题 Schema (FAQPage)
✅ 3. 表格 Schema (Table)
✅ 4. 社交媒体链接 Schema (sameAs)
✅ 5. 博客文章 Schema (BlogPosting) - 自动生成
✅ 6. 公司组织 Schema (Organization) - 已配置好

═══════════════════════════════════════════════════════════════
                   二、如何使用（超简单）
═══════════════════════════════════════════════════════════════

【核心原则】
在文章开头（两个 --- 之间）填写数据，系统自动生成 Schema！

【基本格式】
---
layout: post
title: "文章标题"
date: 2025-11-20

# 以下字段都是可选的
# 想用哪个填哪个，不用就删掉

images:         ← 图片
  - /assets/images/photo1.jpg

faq:            ← 常见问题
  - q: "问题？"
    a: "回答"

table:          ← 表格/参数
  参数名: "值"

sameAs:         ← 社交媒体
  - https://...
---

文章正文内容...

            三、详细使用说明（每种 Schema）
═══════════════════════════════════════════════════════════════

【1. 图片 Schema】

作用：让 AI 在搜索结果中显示您的图片

写法：
---
images:
  - /assets/images/factory.jpg
  - /assets/images/products.jpg
  - /assets/images/team.jpg
---

说明：
- 可以添加 1-10 张图片
- 图片必须先上传到 assets/images/
- 路径必须以 /assets/images/ 开头

自动生成：
{
  "@type": "BlogPosting",
  "image": [
    "https://www.anvsocks.com/assets/images/factory.jpg",
    "https://www.anvsocks.com/assets/images/products.jpg"
  ]
}

───────────────────────────────────────────────────────────────

【2. FAQ Schema（常见问题）】

作用：AI 会在搜索结果中直接显示问答

写法：
---
faq:
  - q: "最小起订量是多少？"
    a: "1000双，可混色混款"

  - q: "交货期多久？"
    a: "现货7天，定制15-20天"

  - q: "支持OEM吗？"
    a: "支持，可定制LOGO和包装"

  - q: "有哪些认证？"
    a: "BSCI, ISO9001, SQP, SCAN, SEDEX 4P"
---

说明：
- q: 后面写问题
- a: 后面写回答
- 可以添加 2-10 个问答
- 问题和回答用引号包起来
- 注意冒号后面有空格

自动生成：
{
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "最小起订量是多少？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "1000双，可混色混款"
      }
    }
  ]
}

───────────────────────────────────────────────────────────────

【3. Table Schema（表格/规格参数）】

作用：AI 提取结构化数据显示

写法：
---
table:
  材质: "70% 美利奴羊毛, 25% 尼龙, 5% 氨纶"
  重量: "50克/双"
  尺寸: "均码 (US 6-11)"
  颜色: "5种（黑/灰/蓝/红/白）"
  包装: "OPP袋或定制礼盒"
  产能: "年产3000万双"
  工厂面积: "3000平方米"
  员工数量: "100+"
  认证: "BSCI, ISO9001, SQP"
---

说明：
- 参数名: "值" 的格式
- 冒号后面有空格
- 值要用引号包起来
- 可以添加任意多个参数

常用参数示例：
  材质:
  尺寸:
  重量:
  颜色:
  包装:
  产能:
  工厂面积:
  员工数量:
  认证:
  起订量:
  交货期:
  价格:

自动生成：
{
  "@type": "Table",
  "data": {
    "材质": "70% 美利奴羊毛",
    "产能": "年产3000万双"
  }
}

───────────────────────────────────────────────────────────────

【4. sameAs Schema（社交媒体/其他网站）】

作用：告诉 AI 您在其他平台的链接

写法：
---
sameAs:
  - https://www.alibaba.com/anvsocks
  - https://www.linkedin.com/company/anvsocks
  - https://www.facebook.com/anvsocks
  - https://www.instagram.com/anvsocks
---

说明：
- 每个链接单独一行
- 行首是 - 和空格
- 必须是完整的 https:// 开头的链接

常用平台：
  - Alibaba（阿里巴巴）
  - LinkedIn（领英）
  - Facebook
  - Instagram
  - Twitter
  - YouTube
  - Made-in-China
  - Global Sources

自动生成：
{
  "@type": "Organization",
  "sameAs": [
    "https://www.alibaba.com/anvsocks",
    "https://www.linkedin.com/company/anvsocks"
  ]
}

                    七、注意事项
═══════════════════════════════════════════════════════════════

【格式要求】

1. 缩进
   ✅ 正确：
   faq:
     - q: "问题？"
       a: "回答"

   ❌ 错误：
   faq:
   - q: "问题？"
   a: "回答"

2. 引号
   ✅ 正确：
   table:
     材质: "70% 羊毛"

   ❌ 错误：
   table:
     材质: 70% 羊毛

3. 冒号后空格
   ✅ 正确：q: "问题？"
   ❌ 错误：q:"问题？"

4. 列表符号
   ✅ 正确：- /assets/images/photo.jpg
   ❌ 错误：/assets/images/photo.jpg

【常见错误】

❌ 图片路径写错
   - 必须是 /assets/images/xxx.jpg
   - 不能是 assets/images/xxx.jpg（缺少开头的 /）

❌ 引号不配对
   - "问题？"  ← 正确
   - "问题？   ← 错误（缺少结尾引号）

❌ 缩进不对齐
   - 用空格缩进，不要用 Tab
   - 每级缩进 2 个空格
