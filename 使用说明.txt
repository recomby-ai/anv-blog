【第一次使用 - 只需做一次】

1. 上传项目到 GitHub
   - 登录 GitHub.com
   - 点击右上角 "+" → "New repository"
   - 输入名字（例如：anv-blog）
   - 点击 "Create repository"
   - 把所有文件上传上去

2. 启用 GitHub Pages（让网站上线）
   - 进入仓库（repository）
   - 点击 "Settings"（设置）
   - 左侧找到 "Pages"
   - Source 选择 "main" 分支
   - 点击 "Save"
   - 等待 5 分钟，网站就上线了！

3. 查看网站地址
   - 还是在 Pages 页面
   - 会显示：Your site is live at https://xxx.github.io
   - 这就是您的博客地址！

═══════════════════════════════════════════════════════════════
                    第二部分：如何发布文章
                   （在 GitHub 网页上操作）
═══════════════════════════════════════════════════════════════

【方法一：复制示例文件（最简单！）】

1. 打开示例文件
   - 进入 GitHub 仓库
   - 点击 "_posts" 文件夹
   - 点击 "EXAMPLE-how-to-write-post.md"
   - 点击右上角"复制"按钮（复制全部内容）

2. 创建新文件
   - 返回 "_posts" 文件夹
   - 点击 "Add file" → "Create new file"
   - 文件名格式：2025-11-20-your-title.md
     ⚠️ 必须以日期开头（年-月-日）
     ⚠️ 中间用 - 连接
     ⚠️ 必须以 .md 结尾

   示例文件名：
   ✅ 2025-11-20-new-products.md
   ✅ 2025-12-01-winter-collection.md
   ✅ 2026-01-15-factory-news.md

3. 粘贴并修改内容
   - 粘贴刚才复制的内容
   - 修改开头部分（在两个 --- 之间）：

   ---
   layout: post
   title: "新产品发布"              ← 改成您的标题
   date: 2025-11-20                  ← 改成今天的日期
   author: ANV Team                  ← 可以改成您的名字
   tags: [产品, 新品, 袜子]          ← 改成相关标签
   category: 产品动态                ← 改成分类
   description: "简短描述"           ← 1-2句话描述
   images:                           ← 图片路径（稍后说明）
     - /assets/images/photo1.jpg
     - /assets/images/photo2.jpg
   ---

4. 写文章内容
   - 在 --- 下面写您的内容
   - 就像写普通文字一样
   - 可以用这些格式：

   # 大标题
   ## 中标题
   ### 小标题

   **粗体文字**
   *斜体文字*

   - 列表项 1
   - 列表项 2

   [链接文字](https://www.anvsocks.com)

5. 保存发布
   - 滚动到页面底部
   - 点击 "Commit new file"（绿色按钮）
   - 完成！5分钟后文章就上线了

                    第三部分：如何添加图片
                   （在 GitHub 网页上操作）
═══════════════════════════════════════════════════════════════

【步骤 1：上传图片到 GitHub】

1. 准备图片
   - 用手机或相机拍照
   - 重命名为简单的英文名字
   - 例如：factory.jpg, products.jpg, team.jpg

   ⚠️ 注意：
   - 不要用中文名字
   - 不要用空格
   - 用小写字母
   - 用 .jpg 或 .png 结尾

2. 上传到 GitHub
   - 进入您的 GitHub 仓库
   - 点击 "assets" 文件夹
   - 点击 "images" 文件夹
   - 点击 "Add file" → "Upload files"
   - 把图片拖到页面上
   - 点击底部 "Commit changes"（绿色按钮）

【步骤 2：在文章中引用图片】

方法 A：在文章开头声明（推荐！）

在文章开头（两个 --- 之间）写：

   images:
     - /assets/images/factory.jpg
     - /assets/images/products.jpg
     - /assets/images/team.jpg

   ✅ 这样图片会自动出现在 AI 搜索结果中！
   ✅ 系统会自动生成 Schema 标记
   ✅ 不需要其他操作

方法 B：在文章内容中插入图片

在文章正文中写：

   ![图片描述](/assets/images/factory.jpg)

   例如：
   ![工厂外观](/assets/images/factory-exterior.jpg)
   ![生产线](/assets/images/production-line.jpg)

【什么是 AI 搜索优化？】

当有人在 ChatGPT、Perplexity、Claude 问：
"告诉我关于诸暨袜子制造商的信息"

AI 会找到您的博客，并显示：
✅ 公司名称和介绍
✅ 工厂照片
✅ 产品信息
✅ 联系方式
✅ 关键数据（面积、产能等）

【系统如何做到的？】

1. Schema.org 结构化数据
   - 位置：_layouts/default.html 和 post.html
   - 作用：告诉 AI 这是什么内容
   - 自动：从您的文章读取标题、描述、图片

2. 自动图片关联
   - 当您在文章开头写：
     images:
       - /assets/images/photo1.jpg

   - 系统自动生成：
     {
       "@type": "BlogPosting",
       "image": [
         "https://www.anvsocks.com/assets/images/photo1.jpg"
       ]
     }

   - AI 就能读取并显示您的图片！

3. llms.txt 文件
   - 专门给 AI 模型看的文件
   - 包含公司基本信息
   - AI 会优先读取这个文件

【您什么都不用做！】
只要您：
- 在文章开头添加 images:
- 系统会自动生成所有 AI 需要的标记
- 2-4 周后 AI 就能搜到您了

问：文章必须用英文写吗？
答：不一定。可以写中文内容，但文件名建议用英文。
    当前配置是英文，更容易被国际 AI 搜索到。

问：多久发一篇文章？
答：建议每月 1-2 篇。内容质量比数量重要。

问：可以写什么内容？
答：- 新产品发布
    - 工厂新闻
    - 获得认证
    - 客户案例
    - 行业动态
    - 节日促销

问：图片必须添加吗？
答：强烈建议！有图片的文章：
    - 更容易被 AI 搜索到
    - 显示效果更好
    - 看起来更专业

问：文章写错了怎么办？
答：在 GitHub 上：
    1. 点击文件
    2. 点击"编辑"按钮（铅笔图标）
    3. 修改内容
    4. 点击 "Commit changes"

问：可以删除文章吗？
答：可以。在 GitHub 上：
    1. 点击文件
    2. 点击"删除"按钮（垃圾桶图标）
    3. 确认删除

问：图片多大合适？
答：- 宽度：1200-2000 像素
    - 格式：JPG 或 PNG
    - 大小：每张不超过 2MB

问：文件名可以用中文吗？
答：不建议。用英文和数字，避免：
    - 中文字符
    - 空格
    - 特殊符号

问：多久能在 AI 搜索看到？
答：- 网站上线：5-10 分钟
    - AI 爬虫索引：1-2 周
    - 完全可搜索：2-4 周

问：示例文件要删除吗？
答：理解后可以删除 EXAMPLE-how-to-write-post.md
    但建议保留作为参考。

【为什么在文章开头写 images: 就能被 AI 搜到？】

1. 您写的内容：
   ---
   images:
     - /assets/images/factory.jpg
   ---

2. 系统自动转换成（在 _layouts/post.html）：
   <script type="application/ld+json">
   {
     "@type": "BlogPosting",
     "image": "https://www.anvsocks.com/assets/images/factory.jpg"
   }
   </script>

3. AI 爬虫看到这个标记，就知道：
   - 这是一篇博客
   - 有一张图片
   - 图片地址是什么
   - 然后在搜索结果中显示

【Schema.org 是什么？】
- 一种标准化的数据格式
- 搜索引擎和 AI 都认识
- 就像给内容贴标签
- 告诉 AI："这是公司"、"这是产品"、"这是图片"

【您需要做的】
就是在文章开头写：
  images:
    - /assets/images/xxx.jpg

剩下的全自动！
